## 3. The Pinterest
*Des utilisateurs, des images (pins) et des commentaires.*

### Schéma Logique
- **users** : id, name
- **pins** : id, url, user_id (FK vers users)
- **comments** : id, content, user_id, pin_id (FK vers users et pins)

*FK = Foreign key*

**Relations :**
- 1-N : un user peut créer plusieurs pins
- 1-N : un pin peut avoir plusieurs commentaires

### Requêtes de création (SQL)

```sql
CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT);

-- Table des images postées par les utilisateurs
CREATE TABLE pins (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  url TEXT,
  user_id INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Table des commentaires sur les images
CREATE TABLE comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  content TEXT,
  user_id INTEGER,
  pin_id INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (pin_id) REFERENCES pins(id)
);
