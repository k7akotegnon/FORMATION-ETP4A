## 1. La Startup de Blogging
*Ce projet lie des utilisateurs à des articles, avec des catégories et des tags.* 

### Schéma Logique
- **users** : id, name
- **articles** : id, title, content, user_id (FK vers users)
- **categories** : id, title
- **article_categories** : article_id, category_id (Table de jointure N-N)
- **tags** : id, title, color
- **category_tags** : category_id, tag_id (Table de jointure N-N)

*FK = Foreign key*

**Relations :**
- 1-N : un user peut créer plusieurs articles (et un article appartient à un seul user).
- N-N : un article peut appartenir à plusieurs catégories (et une catégorie peut avoir plusieurs articles).
- N-N : une catégorie peut avoir plusieurs tags (et un tag peut être lié à plusieurs catégories)

### Requêtes de création (SQL)
```sql
-- Création de la table des auteurs
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT, 
  name TEXT
);

-- Création des articles liés à un auteur (Relation 1-N)
CREATE TABLE articles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT,
  content TEXT,
  user_id INTEGER,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Table des catégories
CREATE TABLE categories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT
);

-- Table de jointure entre Articles et Catégories (Relation N-N)
CREATE TABLE article_categories (
  article_id INTEGER,
  category_id INTEGER,
  FOREIGN KEY (article_id) REFERENCES articles(id),
  FOREIGN KEY (category_id) REFERENCES categories(id)
);

-- Table des tags
CREATE TABLE tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT,
  color TEXT
);

-- Table de jointure entre Catégories et Tags (Relation N-N)
CREATE TABLE category_tags (
  category_id INTEGER,
  tag_id INTEGER,
  FOREIGN KEY (category_id) REFERENCES categories(id),
  FOREIGN KEY (tag_id) REFERENCES tags(id)
);
